## 冒頭の挨拶（つかみ）

> 最初のひとこと。緊張をほぐし、ゴールと進め方を 1 分で。読み上げても、要点だけ拾ってもOK。

- **歓迎** — 「本日はお集まりいただきありがとうございます」
- **名乗り** — 名前／普段の業務をひとことで
- **今日のゴール** — 「“答えるだけの AI”ではなく、**調べて・書いて・保存までやってくれる「動く AI」（Claude Code）を、自分の仕事の道具にする**」
- **進め方** — 3 セッション構成／**見るだけでなく実際に手を動かす**（ハンズオン中心）
- **安心材料** — 失敗してOK・**元に戻せる仕組みがある**／分からなければその場で気軽に質問を
- **環境の確認** — 事前準備（インストール・エディタ・ターミナル）が済んでいるか軽く確認
- **入り** — 「では“答える AI”と“動く AI”の違いから始めます」（→ Session 1）

## トークスクリプト（話す順番・要点）

> 詳細台本ではなく **話す順番と要点のメモ**。基本は画面（デモ＋受講者ページ）を出しながら、「次に何を話す／見せるか」を思い出す用。詰まったら要点だけ言って次へ。【ワーク】= 受講者が手を動かすところ。

### Session 1（20分・概念のみ）

1. **つかみ** — 今日のゴール＝「“動く AI”を仕事の道具にする」。Session 1 は概念だけ・手は動かさない
2. **答える→動く（§1）** — 従来＝答えて終わり／エージェント＝調べて・書いて・保存までやる。表 1 枚で
3. **従来型 vs エージェント（§2-3）** — 比較表で対比。キーワード「自分で段取りして実行」
4. **4 観点→3 キーワード（§4）** — 自立性・連続性・実用性。たとえ：検索エンジン（答える）vs 部下（動く）
5. **Claude Code へ（§5）** — これが“動く AI”。次の Session 2 で実際に触る
6. **質疑 → Session 2 へ**

### Session 2（129分・基礎＋ハンズオン）

1. **オープニング（5分）** — 全体像、事前準備（インストール／エディタ／ターミナル）の確認
2. **Claude Code とは（§1）** — CLI 前提。チャット AI との違いは要点のみ
3. **安全ルール（§2）** — 機密は入れない／ダミーで練習／ファイル名にも注意
4. **許可・停止（§3-4）** — 許可ダイアログ Yes/No、Esc で止まる。実画面は §14 で初登場と予告
5. **チェックポイント（§5）** — 編集前に自動保存＝戻せる、で安心感
   - → **`講義サンプル.zip` 取得・`講義サンプル` フォルダを開く**。ルートに CLAUDE.md／自己紹介.md／議事メモ.txt が見えれば OK
6. **設定を実物で（§6-13）** — CLAUDE.md＝常時ルール／`.claude/`／Skills（2 つ）／MCP はさらっと。`/model` の選び方も要点だけ
7. **【ワーク】自己紹介の穴埋め（§14）** — 【　】を埋める。許可ダイアログ初登場
8. **【ワーク】議事メモ要約＋ルール追記（§15-16）** — `meeting-summary` Skill で `議事録サマリー.md`／CLAUDE.md に 1 行追記
9. **【ワーク】実用ワーク（§17）** — `email-draft` Skill で連絡メール、ToDo 抽出。**デモ中心で手早く**
10. **振り返り** — コマンドリファレンスでおさらい
11. **質疑（15分）** — 想定 Q&A も参照

### Session 3（60分・インストール中心）

1. **振り返り（3分）** — 安全ルール再確認
2. **AI 秘書とは＋ cc-secretary インストール** — 何ができるか→入れる
3. **オンボーディング・基本操作** — デモ中心
4. **自分仕様に育てる／代替ワーク** — ここで cc-secretary 完了
5. **Lark CLI インストール・設定・Features** — 環境差で前後しやすい。詰まったら短縮

## 読み方ガイド

講義中に口に出す機会が多く、スペルから読み方が分かりにくい用語の一覧です。スペル通りにアルファベットを読むだけのもの（CLI、API、MCP など）は省いています。

| 表記 | 読み方 | 補足 |
|---|---|---|
| `cc-secretary` | シーシー・セクレタリー | cc = Claude Code の略 |
| `npm` | エヌ・ピー・エム | 「ナム」とは読まない。Node Package Manager の略 |
| `npx` | エヌ・ピー・エックス | npm に同梱。パッケージを入れずに 1 回だけ実行するツール |
| `Node.js` | ノード・ジェイエス | .js = JavaScript の略称 |
| `JSON`（settings.json など） | ジェイソン | JavaScript Object Notation の略 |
| `bash` | バッシュ | Bourne Again Shell。ターミナルのシェル名 |
| `zsh` | ズィーシェル | Z Shell。macOS のデフォルトシェル |
| `Anthropic` | アンソロピック | Claude の開発元。「アンスロピック」と誤読されやすい |
| `Antigravity` | アンチグラビティ | 講義で使う IDE の 1 つ |
| `Lark` | ラーク | Lark Suite。Session 3 の Lark CLI で頻出 |

## 許可ダイアログの選択肢（読み方・受講者への説明）

受講者ページ §3 の許可ダイアログに出る英語の選択肢。講義中に読み上げて意味を説明する想定（実際に画面で初めて出るのは **§14** のファイル編集時）。

| 表記 | 読み方 | 受講者への説明 |
|---|---|---|
| `Yes` | イエス | 今回の操作だけ許可。基本はこれを選ぶ |
| `Yes, allow all edits during this session` | イエス、アロウ・オール・エディッツ・デュアリング・ディス・セッション | このセッション中の編集をすべて許可（以後ダイアログが出なくなる）。頻出操作に慣れてから検討 |
| `No` | ノー | 拒否。実画面では `No, and tell Claude what to do differently`（ノー、アンド・テル・クロード・ホワット・トゥ・ドゥ・ディファレントリー＝拒否して別の指示を出す）と続く |

> [!NOTE]
> 「セッション中」= `claude` を起動してから終了（`/exit`）するまで。終了すると許可はリセットされ、次回起動時にまた確認が出ます。

## 想定 Q&A（受講者から来そうな質問）

質疑応答（Session 2 進行表の最終枠・15 分）や実習中に出やすい質問と、講師の回答例。**節番号は受講者ページの節**を指す。回答は要点のみ——詳細は該当節へ誘導する。

### 料金・プラン

| 質問 | 回答（要点） |
|---|---|
| お金はかかる？ | 本講義は **Claude Pro プラン以上** のサブスクが前提。Anthropic 公式のトークン単価表（API 課金）は別の仕組みで、Claude Code の個人利用には当てはまらない。 |
| 使いすぎると止まる？ | レート制限あり。上限に近づくと表示が出る。`/context` で消費量、`/compact`・`/clear` で会話を整理。目安は **付録 B**。 |
| 無料で試せる？ | 無料枠は限定的。本講義の手順は Pro 以上を想定。 |

### 安全・プライバシー（§2）

| 質問 | 回答（要点） |
|---|---|
| 入力内容は学習に使われる？外に出る？ | 実行は手元の PC だが、**プロンプトや読み込んだファイルの内容は Anthropic に送信**される。機密情報は入れない（§2）。 |
| 勝手にファイルを消さない？ | 編集の前に **許可ダイアログ** が出る（§3）。編集前に **チェックポイント** が作られ元に戻せる。基本は `Yes`、不安なら `No`。 |
| 会社の機密ファイルで使っていい？ | 講義中は **ダミーのみ**。実務利用は社内ルールに従い、ファイル名・コミットに機密を含めない。 |

### 環境・インストール（事前準備）

| 質問 | 回答（要点） |
|---|---|
| Windows でも動く？ | 動く。Mac / Windows、CLI / IDE 拡張で利用可。 |
| エディターは何を使う？ | VS Code / Cursor / Antigravity のいずれか（事前準備 §②）。 |
| ターミナルが不安 | 本講義でやるのは「内蔵ターミナルで `claude` を起動」だけ（事前準備 §③）。 |
| Desktop アプリでもいい？ | 同じエンジンなので可。ただし本講義は **CLI（ターミナル版）前提** で進める。 |

### 操作でつまずいたら（受講者ページ末尾の「トラブルシューティング」も参照）

| 症状・質問 | 対処 |
|---|---|
| 日本語で指示していい？ | OK。普通の日本語で頼んでよい。 |
| 編集されない／ダイアログが出ない | 開いているフォルダ違いが多い。**`講義サンプル` フォルダ** を開き直す。 |
| 応答が遅い・止まらない | **Esc で中断**（§4）。長くなったら `/context` → `/compact` か `/clear`。 |
| モデルは何を選ぶ？ | 迷ったら **Sonnet**。複雑なら Opus、速さ優先なら Haiku（`/model` の選び方は §13）。 |
| `CLAUDE.md` と Skill の違いは？ | `CLAUDE.md`＝常に効くルール（起動時に自動読込）。Skill＝必要なときだけ呼ぶ定型手順（§6・§11）。 |

### 講義サンプル（今回の仕様）

| 質問 | 回答（要点） |
|---|---|
| DL したファイル名／フォルダ名が資料と違う？ | ダウンロードは **`講義サンプル.zip`**、解凍で **`講義サンプル`** フォルダ。古いブラウザでは `sample-data.zip` で落ちることがあるが中身は同じ。 |
| 解凍したら文字化けする | Finder / エクスプローラーでの解凍は問題なし。ターミナルの `unzip -l` の **表示だけ** 化ける場合がある（実体は正常）。 |
| `CLAUDE.md` などを日本語名にできない？ | できない。**Claude Code が決めた固定名** で、変えると認識されない（`説明書.md` 参照）。 |
| `議事録サマリー.md` が別名で保存される | プロンプトで **ファイル名を明示** する。Skill（meeting-summary）経由なら `議事録サマリー.md` を指定済み。 |

### Session 3（AI 秘書・Lark）— Session 2 で聞かれたら

| 質問 | 回答（要点） |
|---|---|
| AI 秘書（cc-secretary）って？ | タスク・メモ・予定を Claude Code 上で管理する仕組み。**Session 3 で扱う** ので Session 2 では深入りしない。 |
| Lark CLI は必須？ | Session 3 の題材。**Lark アカウントが無くても代替ワークあり**（Session 3 §7）。 |

> [!NOTE]
> 講義サンプルには AI 秘書・Lark の話題を**あえて入れていない**（Session 3 で初出にするため）。Session 2 で深掘り要望が出たら「Session 3 で扱う」と案内して切り上げる。

## サンプルデータ（開発・講師環境）

受講者への配布は **講義サイト** の `/api/sample-data`（**Session 2** の **§5 直後**に表示されるダウンロードボタン）のみ。受講者がダウンロードする ZIP 名は **`講義サンプル.zip`**、解凍して開くフォルダ名も **`講義サンプル`**（ZIP 内ルート＝`lib/sample-data-zip.ts` の prefix）。ZIP 名は `app/api/sample-data/route.ts` の `Content-Disposition` で UTF-8（RFC 5987）指定し、古いクライアント向けに ASCII フォールバック `sample-data.zip` も併記している。一方で **API の URL `/api/sample-data` とリポジトリ内のフォルダ名 `sample-data` は ASCII のまま**（macOS の NFD/NFC 正規化による git の幽霊 diff を避けるため）。リポジトリを clone 済みの講師は次のパスを直接開ける

```bash
# エディターで開く
docs/training/claude-code-personal/sample-data

# またはホームへコピー
cp -r docs/training/claude-code-personal/sample-data ~/claude-code-personal
```
