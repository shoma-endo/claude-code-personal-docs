# 講義用サンプルデータ（ワークスペース）

Claude Code 個人向け 1Day 講義の演習用フォルダです。**エディターで開くのはこの `講義サンプル` フォルダ**（`CLAUDE.md` がルートに見える階層）です。実在の顧客名・個人名・金額は含みません。

## 取得方法

講義サイト（**Session 2** の **§5 のあと** — 講義用サンプルデータの準備）の **ダウンロードボタン** から **`講義サンプル.zip`** を取得し、展開してください（展開すると **`講義サンプル`** フォルダが出てきます）。**§6 より前に**完了してください。

## エディターで開く

1. **`講義サンプル` フォルダ** を VS Code / Cursor / Antigravity IDE で「フォルダを開く」
2. 内蔵ターミナル（`` Ctrl + ` ``）を開いて `claude` を起動（事前準備で済ませておく）

```bash
claude
```

> [!IMPORTANT]
> ルートに `CLAUDE.md`・`自己紹介.md`・`議事メモ.txt` が並んで見えていれば正しい階層です。

## フォルダ構成

```text
講義サンプル/
├── 説明書.md                   # 最初に読む（このファイル）
├── CLAUDE.md                   # Session 2 §6 以降。プロジェクトルール
├── 自己紹介.md                 # Session 2 §14 用（穴埋めテンプレート・差し替え演習）
├── 議事メモ.txt                # Session 2 §15 用（要約のもとになるダミー議事録）
└── .claude/
    ├── settings.json           # §8 の設定例（中身はほぼ空でOK）
    └── skills/
        ├── meeting-summary/
        │   └── SKILL.md        # §11・§15 の Skill 例（議事メモ要約）
        └── email-draft/
            └── SKILL.md        # §17B の Skill 例（連絡メール下書き・型入り）
```

> [!IMPORTANT]
> **ファイル名には2種類あります。**
> - **日本語名のファイル**（`説明書.md`・`自己紹介.md`・`議事メモ.txt`）— 演習用に**自由に付けた名前**です。中身を読んだり書き換えたりする「素材」です。
> - **英語の固定名**（`CLAUDE.md`・`.claude/`・`settings.json`・`SKILL.md`）— **Claude Code が決めた名前**で、変更できません。Claude Code はこの名前を**目印にして自動で読み込みます**。実務でも全く同じ名前を使うので、英語のまま覚えるのが正解です。
>
> 「日本語にしないの？」と思うかもしれませんが、固定名は**日本語化すると Claude Code が認識できなくなる**ため、あえて本物の名前のまま体験します。

## ダウンロード後にやること（はじめての方向け・順番）

このフォルダを取得したら、上から順に進めれば一通り体験できます。各ステップは講義サイト（受講者ページ）の節と対応しています。

> [!NOTE]
> **共通の設定（ゆるい物語）**：このフォルダの素材は、どれも「**個人で Web 制作・運用をしているあなた**」と、その**協力メンバー（業務委託先など）**の日常業務を題材にしています。まず自己紹介で自分の仕事を整理し（§14）、続いて協力メンバーと行った業務改善ミーティングのメモを要約して成果物にする（§15）——という一連の流れとしても読めます。ただし各ワークは**独立していて、単独でも進められます**（順番は必須ではありません）。

### ステップ0 — 準備

1. この `講義サンプル` フォルダをエディターで「フォルダを開く」
2. 内蔵ターミナル（`` Ctrl + ` ``）で `claude` を起動する

### ステップ1〜 — 進め方

まず各ファイルの中身を軽く確認してから、**手を動かすワーク**に進みます。

#### まず中身を知る（読む・要約させる）

| 確認すること | 使うファイル | 受講者ページ |
|---|---|---|
| プロジェクトルールを Claude に要約させる | `CLAUDE.md` | §6 |
| 設定ファイルの置き場所を知る（中身はほぼ空でOK） | `.claude/settings.json` | §8 |
| Skill（定型手順）の中身を読む | `.claude/skills/meeting-summary/SKILL.md` | §11 |

#### 手を動かすワーク

| ワーク | 使うファイル | 受講者ページ |
|---|---|---|
| 自己紹介テンプレートの【　】を自分用に埋める | `自己紹介.md` | §14 |
| 議事メモを要約して保存する | `議事メモ.txt` → `議事録サマリー.md` | §15 |
| `CLAUDE.md` にルールを 1 行追記する | `CLAUDE.md` | §16 |
| 自分の ToDo を抜き出す／連絡メールを下書きする（`email-draft` Skill） | `議事メモ.txt`・`議事録サマリー.md` | §17 |

> [!TIP]
> 詳しい依頼文（プロンプト）や完了の目安は、受講者ページの各節に載っています。このフォルダは「手を動かす場所」、受講者ページは「進め方の説明」と覚えてください。

## このサンプルでできること（例）

- Claude にファイルを**読ませて要約**させる（議事メモ → サマリー）
- 既存ファイルを**目的に合わせて編集**させる（自己紹介の差し替え）
- **プロジェクトルール（`CLAUDE.md`）**で Claude の応答ルールを決める
- **Skill** で定型作業（議事録整理）を手順化する

## データの出典

議事メモの文体・項目構成は、公開されている議事録サンプル（例: [Sample.net — Minutes of Meeting](https://www.sample.net/minutes-of-meeting)）を参考に、**講義用に新規作成したフィクション**です。
